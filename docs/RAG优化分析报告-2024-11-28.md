## 二、当前架构问题与矛盾

### 🔴 严重问题

**1. AIP聊天topK=300太高，且无reranker**
- 位置：`aip聊天助手-未来教育 探索者联盟.json`
- 问题：返回300个文档片段给LLM，会导致：
  - 上下文过长，LLM容易迷失
  - 不相关内容干扰回答质量
  - 成本高（token消耗大）
- 对比：盖亚聊天topK=5且有reranker，设计合理

**2. 无父子片段（Parent-Child Chunks）**
- 当前两个上传工作流都使用简单的递归文本分割
- 问题：小片段可能丢失上下文，导致检索到的内容不完整

**3. 无混合检索（Hybrid Search）**
- 当前只有向量相似度检索
- 问题：对于精确术语、专有名词匹配效果差

### 🟡 设计矛盾

**1. 分片策略不一致**
- 盖亚知识库：chunk=800, overlap=200
- AIP知识库：chunk=400, overlap=100
- 矛盾：为什么AIP用更小的分片？如果是为了精确，但topK=300又太大

**2. 两套独立系统（合理设计）**
- futuremind库：盖亚课程体系
- aip库：探索者联盟体系
- **说明**：两套系统服务不同的教育场景，保持独立是合理的设计决策

## 三、RAG优化建议

### 优先级1：修复AIP聊天配置
```
当前：topK=300, 无reranker
建议：topK=20-50, 添加Cohere reranker
```

### 优先级2：实现父子片段
```
策略：
- 父片段：1500-2000字符（保留完整上下文）
- 子片段：300-500字符（用于精确检索）
- 检索时：用子片段匹配，返回父片段内容
```

### 优先级3：添加混合检索
```
策略：
- 向量检索：语义相似度 (权重0.7)
- 关键词检索：BM25/全文搜索 (权重0.3)
- 合并结果后rerank
```

### 优先级4：统一分片策略
```
建议：
- 统一chunk=600-800, overlap=150-200
- 根据文档类型调整（课程内容vs项目文档）
```

## 四、盖亚提示词优化建议

当前`心灵学院聊天bot 盖亚.json`中的Agent提示词已经很详细，但可以优化：

### 需要加强的方面

**1. 苏格拉底式提问**
```
当前：有提到"引导式对话"
建议增加：
- 每次回复不超过3-4句话
- 以一个开放性问题结尾
- 避免直接给答案，用问题引导思考
```

**2. 简洁回复**
```
当前：回复可能过长
建议：
- 设置字数限制（100-200字）
- 分段回复复杂问题
- 使用"想知道更多吗？"让用户主动深入
```

**3. 追问机制**
```
建议增加：
- 根据用户回答深度调整追问
- 检测用户情绪，适时切换话题
- 记录用户兴趣点用于后续对话
```

## 十、课程自动解析系统优化（2024-11-29更新）

### 现有系统位置
- 前端页面：`app/admin/courses/new/page.tsx`
- 解析API：`app/api/admin/courses/parse/route.ts`
- 使用模型：Gemini 2.0 Flash

### 现有流程
1. 上传.md/.txt文档
2. AI识别课程类型（listening/earth/pbl/daily）
3. 根据类型选择提示词解析
4. 预览确认后保存到course_contents表

### 已知问题
- 长文档分批处理时内容可能丢失
- AI对分隔符格式理解不稳定
- 知识点等字段可能不完整

### 优化方案

| 方案 | 描述 | 难度 | 效果 |
|------|------|------|------|
| A. 优化提示词 | 增加完整性要求 | 低 | 中等 |
| B. 分步解析 | 逐天/模块单独解析 | 中 | 好 |
| C. 模板+AI | 规则提取结构，AI理解内容 | 高 | 最佳 |
| D. 人工校验 | 增加编辑预览功能 | 低 | 保障质量 |

### 推荐执行顺序
1. 先做D（人工校验）- 保障现有功能可用
2. 再做A（优化提示词）- 提升AI解析准确率
3. 后做B（分步解析）- 解决长文档问题
