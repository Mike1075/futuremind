## 🐛 常见问题排查

### 问题 1: 调用返回 401 错误

**原因**: Authorization token 过期或错误

**解决**:
1. 在 Supabase Dashboard → Settings → API 获取最新的 `anon` key
2. 更新测试脚本中的 `SUPABASE_ANON_KEY`

### 问题 2: 返回 500 错误

**原因**: OpenAI API 调用失败或数据库查询错误

**解决**:
1. 查看 Edge Function 日志：Supabase Dashboard → Functions → summarize-user-activity → Logs
2. 检查 `OPENAI_API_KEY` 环境变量是否正确配置
3. 检查 OpenAI 账户余额

### 问题 3: AI 总结内容为"无法生成总结"

**原因**: GPT-4o-mini API 调用失败

**解决**:
1. 检查 OpenAI API key 是否有效
2. 检查 OpenAI 账户是否有余额
3. 查看 Edge Function 日志中的错误信息

### 问题 4: 没有返回任何总结

**原因**: 用户没有对话、作业或项目数据

**解决**:
1. 选择一个有数据的用户进行测试
2. 执行以下 SQL 查找有数据的用户:

```sql
-- 查找有对话数据的用户
SELECT DISTINCT user_id, COUNT(*) as conversation_count
FROM gaia_conversations
WHERE is_active = true
GROUP BY user_id
ORDER BY conversation_count DESC
LIMIT 5;

-- 查找有作业数据的用户
SELECT DISTINCT user_id, COUNT(*) as submission_count
FROM user_submissions
GROUP BY user_id
ORDER BY submission_count DESC
LIMIT 5;

-- 查找有项目数据的用户
SELECT DISTINCT user_id, COUNT(*) as project_count
FROM user_selected_projects
GROUP BY user_id
ORDER BY project_count DESC
LIMIT 5;
```
