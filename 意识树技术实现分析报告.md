
**触发条件**: `depth_level > 3`

**为什么 depth_level > 3 才有二级根？**
- **设计理由**: 模拟真实植物的根系发育
  - **浅根** (depth_level ≤ 3): 刚开始探索，只有主根
  - **深根** (depth_level > 3): 已经扎根深入，开始分叉寻找养分
- **视觉理由**: 避免 depth_level 很小时产生过多细节

**为什么没有第三级根？**
- ❌ **技术原因**: 代码未实现递归生成，只有1层分支
- ✅ **设计理由**:
  1. 避免视觉过于复杂，难以辨识
  2. 二级根已足够表达"扎根深度"的概念
  3. 性能考虑：三级根会大幅增加粒子数量（每个主根4个末端）

**数学关系**:
```
主根数量: count
二级根数量: count * 2 (每个主根分2个)
总末端数量: count * 3 (1个主根末端 + 2个二级末端)

假设 count=12, depth_level=5:
- 12条主根
- 24条二级根
- 36个末端
```

#### ⚠️ 为什么调整叶子/果实滑块时"只看见枝条在跳动"？

**问题根源**:

1. **枝条位置有随机性** (第275-279行)：
   ```typescript

   const startX = centerX + random(-trunkThickness * 0.5, trunkThickness * 0.5)
   const angle = random(-75, -105) + (i % 2 === 0 ? -30 : 30) + random(-15, 15)
   ```

2. **每次参数变化都完全重新生成整棵树**：
   - 用户调整 `leaves.count` 滑块
   - 触发 React 重新渲染 (`ConsciousnessTreeClient.tsx`)
   - `generateConsciousnessTree()` 被完整调用
   - branches 的随机位置重新计算
   - `branchEndpoints` 数组完全变化
   - 叶子和果实也跟着重新分布

3. **叶子/果实太小，不明显**：
   - 叶子只有 **3 像素**（`particleSize * 1.5`）
   - 果实只有 **6 像素**（`particleSize * 3`）
   - 在黑色背景上，红色小点不如明显的枝条线条

**视觉效果**:
- ✅ 枝条位置剧烈变化 → 非常明显（线条长，变化大）
- ❌ 叶子/果实也在变，但太小看不清 → 用户只注意到"枝条在跳动"

**数学说明**:
```
假设 branches.count = 17, avg_length = 10.0:
- 每次渲染，17条枝条的起点 (startY, startX) 都是随机的
- 角度 angle 也包含 random(-15, 15) 的随机分量
- 末端坐标 = startPoint + length * (cos(angle), sin(angle))
- 17个末端位置完全重新计算
- 叶果从这17个新末端随机选择，也跟着跳动
```

## 三、参数范围设计理由

### 3.1 为什么深度是 0-10？

**参数**: `roots.depth_level` (0-10)

**计算公式** (第191行):
```typescript
const rootLength = Math.max(growthData.roots.depth_level * 15, 5)
```

**设计理由**:
- `depth_level = 10` → `rootLength = 150` 像素
- 假设画布高度 **600px**，baseY = **390px** (65%位置)
- 根系最长可达 390 + 150 = **540px**，接近画布底部
- 如果加上二级根：540 + 75 = **615px**，刚好超出画布
- **结论**: 10 是基于视觉空间计算的合理上限

**扩展说明**:
```
画布高度: 600px
树干底部: 600 * 0.65 = 390px
可用下方空间: 600 - 390 = 210px

主根长度: depth_level * 15
二级根长度: 主根 * 0.5 = depth_level * 7.5
总延伸: depth_level * 22.5

depth_level = 10 → 225px > 210px (刚好接近边界)
depth_level = 9 → 202.5px (合适)
```

## 四、颜色系统详解

### 4.1 颜色计算公式

**位置**: `lib/utils/consciousnessTreeGenerator.ts:56-104`

```typescript
const getColor = (
  type: 'root' | 'trunk' | 'branch' | 'leaf' | 'fruit',
  value: number,    // 归一化值 0-1
  isSolid: boolean,
  glowIntensity: number = 0.5
): string => {
  // 1. 基础参数（根据类型）
  let hue = 0
  let baseLightness = 25
  let baseSaturation = 60

  switch (type) {
    case 'root':   hue=0, baseLightness=25, baseSaturation=60; break
    case 'trunk':  hue=0, baseLightness=30, baseSaturation=65; break
    case 'branch': hue=0, baseLightness=40, baseSaturation=70; break
    case 'leaf':   hue=5, baseLightness=45, baseSaturation=75; break  // 稍带金感
    case 'fruit':  hue=8, baseLightness=50, baseSaturation=80; break  // 金边红
  }

  // 2. 动态计算（根据数值）
  const lightness = baseLightness + value * (72 - baseLightness)
  const saturation = baseSaturation + value * (100 - baseSaturation)

  // 3. 透明度控制
  const baseAlpha = isSolid ? 0.6 : 0.25
  const alpha = baseAlpha * glowIntensity

  return `hsla(${hue}, ${saturation}%, ${lightness}%, ${alpha})`
}
```

### 4.2 颜色梯度示例表

| 类型 | value=0 (最小) | value=0.5 (中等) | value=1.0 (最大) |
|------|----------------|------------------|------------------|
| **Root** | `hsla(0, 60%, 25%, 0.15)` 深暗红 | `hsla(0, 80%, 48.5%, 0.15)` 中等红 | `hsla(0, 100%, 72%, 0.15)` 亮红 |
| **Trunk** | `hsla(0, 65%, 30%, 0.15)` 深红 | `hsla(0, 82.5%, 51%, 0.15)` 鲜红 | `hsla(0, 100%, 72%, 0.15)` **亮红(发白)** |
| **Branch** | `hsla(0, 70%, 40%, 0.36)` 标准红 | `hsla(0, 85%, 56%, 0.36)` 亮红 | `hsla(0, 100%, 72%, 0.36)` **极亮红** |
| **Leaf** | `hsla(5, 75%, 45%, 0.15)` 金红暗 | `hsla(5, 87.5%, 58.5%, 0.15)` 金红亮 | `hsla(5, 100%, 72%, 0.15)` 金白 |
| **Fruit** | `hsla(8, 80%, 50%, 0.15)` 金边红 | `hsla(8, 90%, 61%, 0.15)` 亮金红 | `hsla(8, 100%, 72%, 0.15)` **金白** |

**注**: 透明度 = `isSolid ? 0.6 : 0.25` (虚体时减半)

### 4.3 树干"发白"问题深度分析

**问题**: 当 `height_level = 83` 时，树干颜色接近白色

**数学推导**:
```typescript
// 树干的颜色计算 (第170-175行)
const trunkColor = getColor('trunk', height_level / 100, is_solid, 0.5)

// 代入 height_level = 83
value = 83 / 100 = 0.83

// 计算明度 (第96行)
lightness = 30 + 0.83 * (72 - 30)
          = 30 + 0.83 * 42
          = 30 + 34.86
          = 64.86%

// 计算饱和度 (第97行)
saturation = 65 + 0.83 * (100 - 65)
           = 65 + 0.83 * 35
           = 65 + 29.05
           = 94.05%

// 最终颜色
hsla(0, 94.05%, 64.86%, 0.3)  // 高明度 + 高饱和度 = 接近粉红/白色
```

**为什么会发白？**
- **明度过高**: 64.86% 已经超过中间值 50%，进入"亮色"区间
- **HSL色彩空间特性**: 当明度 > 60% 时，任何颜色都会趋向白色
- **红色系特殊性**: 红色在高明度下比其他颜色更容易发白

**标准建议**:
```
深色: L = 20-40%  (暗红、深红)
中色: L = 40-55%  (标准红、鲜红)
亮色: L = 55-65%  (亮红、粉红) ⚠️
过亮: L = 65-80%  (接近白色) ❌
```

### 5.3 ⚠️ 问题3: 没有第三级根/枝

**当前实现**:
- 根系: 最多2级（主根 + 二级根）
- 枝条: 最多2级（主枝 + 二级枝）

**限制原因**:
- 代码采用手动控制的分支逻辑，未使用递归
- 第225行和第299行硬编码了二级分支生成

**如果需要三级分支，建议实现递归系统**:

```typescript
// 新增递归函数
function generateBranches(
  particles: Particle[],
  startPoint: { x: number, y: number },
  direction: number,     // 角度
  length: number,        // 长度
  width: number,         // 宽度
  depth: number,         // 当前深度
  maxDepth: number,      // 最大深度
  color: string,
  isSolid: boolean,
  particleSize: number
) {
  if (depth > maxDepth || length < 5) return

  // 计算终点
  const endX = startPoint.x + length * Math.cos((direction * Math.PI) / 180)
  const endY = startPoint.y + length * Math.sin((direction * Math.PI) / 180)

  // 绘制当前分支
  drawLine(particles, startPoint.x, startPoint.y, endX, endY, width, 'branch', color, isSolid, particleSize)

  // 递归生成子分支（1-3个）
  const subLength = length * 0.6
  const subWidth = width * 0.7

  generateBranches(particles, {x: endX, y: endY}, direction - 30, subLength, subWidth, depth + 1, maxDepth, color, isSolid, particleSize)
  generateBranches(particles, {x: endX, y: endY}, direction + 30, subLength, subWidth, depth + 1, maxDepth, color, isSolid, particleSize)
}
```

**性能考虑**:
- 二级分支: `count * 2` (假设20条 → 40个末端)
- 三级分支: `count * 4` (假设20条 → 80个末端)
- 四级分支: `count * 8` (假设20条 → 160个末端)
- **建议**: 最多3级，避免粒子数量爆炸

### 6.3 虚实线绘制原理

**位置**: `lib/utils/consciousnessTreeGenerator.ts:111-147`

**核心算法**:
```typescript
const drawLine = (
  particles: Particle[],
  x1: number, y1: number,
  x2: number, y2: number,
  width: number,
  type: 'root' | 'trunk' | 'branch',
  color: string,
  isSolid: boolean,
  particleSize: number
) => {
  const dist = Math.hypot(x2 - x1, y2 - y1)

  // 关键：虚实线的粒子间距差异
  const stepSize = isSolid ? (particleSize * 0.4) : (particleSize * 3.0)
  const steps = Math.max(Math.floor(dist / stepSize), 1)

  // 根据宽度计算并列层数
  const layers = Math.max(1, Math.floor(width / (particleSize * 0.8)))

  for (let i = 0; i <= steps; i++) {
    const t = i / steps
    const baseX = x1 + (x2 - x1) * t
    const baseY = y1 + (y2 - y1) * t

    // 在宽度范围内并列分布粒子
    for (let layer = 0; layer < layers; layer++) {
      const offset = (layer / Math.max(layers - 1, 1) - 0.5) * width
      const angle = Math.atan2(y2 - y1, x2 - x1) + Math.PI / 2
      const px = baseX + offset * Math.cos(angle) + (Math.random() - 0.5) * 1
      const py = baseY + offset * Math.sin(angle) + (Math.random() - 0.5) * 1

      particles.push({ x: px, y: py, size: particleSize, color })
    }
  }
}
```

**实现原理**:
1. **实线** (`isSolid = true`):
   - 粒子间距 = `particleSize * 0.4` = 0.8px
   - 粒子密集排列 → 形成连续光带

2. **虚线** (`isSolid = false`):
   - 粒子间距 = `particleSize * 3.0` = 6px
   - 粒子稀疏分布 → 形成点状虚线

3. **宽度控制**:
   - 计算需要多少层粒子并列（`layers`）
   - 在垂直于线条的方向上均匀分布

## 十、总结

### 核心设计理念

1. **累积生长制**: 所有参数只增不减，记录用户成长历程
